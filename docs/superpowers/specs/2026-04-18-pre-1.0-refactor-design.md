# Pre-1.0 Refactor Design

**Date:** 2026-04-18

## Overview

Two-phase internal refactor before the 1.0 release. Phase 1 is purely structural (zero observable behaviour change). Phase 2 changes two observable behaviours and requires acceptance test review.

`TransformerOptions` (the exported public API) is unchanged in both phases.

---

## Phase 1 — Structural

### 1. `TransformerContext` object

Replace the 8–11 individual parameters that flow from `transformSourceFile` → `visitNode` → `tryRewriteFunction` / `tryRewriteClass` → all sub-functions with a single context object.

**Definition (internal to `transformer.ts` or a new `transformer-context.ts`):**

```ts
type TransformerContext = {
  warn: (msg: string) => void;
  checker: typescript.TypeChecker | undefined;
  allowIdentifiers: string[];
  keepContracts: KeepContracts;
  paramMismatch: ParamMismatchMode;
  reparsedIndex: ReparsedIndex;            // set per file in transformSourceFile
  reparsedCache: Map<string, typescript.SourceFile>;
  transformed: { value: boolean };         // per-file mutation flag
};
```

`transformSourceFile` constructs the context, overrides `keepContracts` from the file directive, then passes `ctx` down. All internal functions take `ctx` instead of individual args. `TransformerOptions` (exported) is unchanged.

### 2. New source files from splitting `function-rewriter.ts`

`function-rewriter.ts` is 661 lines mixing four unrelated concerns. Split as follows:

| File | Responsibility | Approx lines |
|---|---|---|
| `keep-contracts.ts` | `KeepContracts` type, `shouldEmitPre/Post/Invariant`, `allContractsFiltered` | ~70 |
| `contract-utils.ts` | `expressionUsesResult`, `expressionUsesPrev`, shared `KIND_PRE`/`KIND_POST`/`PREV_ID` constants | ~50 |
| `tag-pipeline.ts` | `filterValidTags`, `filterPostTagsWithResult`, `filterPostTagsRequiringPrev`, `resolvePrevCapture`, `extractAndFilterTags`, `buildTagInputs` | ~200 |
| `function-rewriter.ts` (trimmed) | `normaliseArrowBody`, `applyNewBody`, `buildGuardedStatements`, scope analysis, `rewriteFunction`, `tryRewriteFunction` | ~300 |

### 3. Remove duplication in `class-rewriter.ts`

`class-rewriter.ts` currently:
- Redeclares `expressionUsesPrev` (identical to the one in `function-rewriter.ts`)
- Redeclares `KIND_PRE`, `KIND_POST`, `PREV_ID` constants

Both are removed and imported from `contract-utils.ts` instead.

### Acceptance test impact — Phase 1

None. All changes are internal restructuring with no behaviour change.

## Phase 2 — Cleanup

### 1. Fix silent `catch {}` in `tryRewriteFunction` and `tryRewriteClass`

Both functions currently swallow all errors silently. Change to surface them via `warn` while still returning the original node (compilation continues):

```ts
// before
} catch {
  return node;
}

// after — in tryRewriteFunction
} catch (e) {
  ctx.warn(
    `[axiom] Internal error in ${buildLocationName(node)}: ${e instanceof Error ? e.message : String(e)}`
  );
  return node;
}

// after — in tryRewriteClass
} catch (e) {
  ctx.warn(
    `[axiom] Internal error in ${node.name?.text ?? 'UnknownClass'}: ${e instanceof Error ? e.message : String(e)}`
  );
  return node;
}
```

### 2. Unify the two unsupported-function warning helpers in `transformer.ts`

`emitUnsupportedClosureWarning` (line 98) and `emitUnsupportedExpressionWarning` (line 112) emit the same message format with different name-extraction logic. Merge into one helper:

```ts
function emitUnsupportedFunctionWarning(name: string, warn: (msg: string) => void): void {
  warn(
    '[axiom] Warning: @pre/@post on arrow functions, function expressions, and closures'
    + ` is not supported — contracts were not injected (in ${name})`,
  );
}
```

Message text is unchanged — no acceptance test updates needed.

### 3. Simplify `shouldEmit*` in `keep-contracts.ts`

Replace the verbose if-chain bodies with one-liners:

```ts
export const shouldEmitPre       = (k: KeepContracts): boolean => k === false || k === 'pre'       || k === 'all';
export const shouldEmitPost      = (k: KeepContracts): boolean => k === false || k === 'post'      || k === 'all';
export const shouldEmitInvariant = (k: KeepContracts): boolean => k === false || k === 'invariant' || k === 'all';
```

No behaviour change.

### 4. Fix `mergeIdentifiers` side-effect smell in `function-rewriter.ts`

`mergeIdentifiers` currently mutates `preKnown`/`postKnown` AND returns `exportedNames`. Split into two explicit operations:

```ts
// enrichKnownIdentifiers — mutates the sets, returns void
function enrichKnownIdentifiers(
  preKnown: Set<string>,
  postKnown: Set<string>,
  checker: typescript.TypeChecker | undefined,
  node: typescript.FunctionLikeDeclaration,
  allowIdentifiers: string[],
): void

// collectExportedNames — already exists, called separately and assigned explicitly
const exportedNames = collectExportedNames(node.getSourceFile());
```

No behaviour change.

### Acceptance test impact — Phase 2

| Change | Impact |
|---|---|
| `catch {}` → `warn(error)` | Previously silent errors now surface as warnings. After implementing, grep acceptance tests for warning-count assertions on error paths. |
| Warning unification | None — message text unchanged. |
| `shouldEmit*` simplification | None. |
| `mergeIdentifiers` split | None. |
